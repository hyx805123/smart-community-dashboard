# 智慧社区监控大屏 — 优化说明技术文档

## 一、项目概述

本文档说明将原纯 HTML 智慧社区监控大屏项目，从商用角度进行深度优化的全过程。优化目标包括：消除代码冗余、修复现有缺陷、引入现代前端工程化架构、增加商用级功能（搜索筛选、分页、导出、权限等），并提供完整的 Mock 数据演示环境与真实 API 预留接口。

---

## 二、原项目问题分析

### 2.1 代码结构问题

| 问题 | 影响 |
|------|------|
| 10 个 HTML 页面各自复制完整的 sidebar 和 CSS | 任何侧边栏修改需要改 10 处，维护成本极高 |
| 无组件化拆分 | 统计卡片、图表、表格逻辑高度重复 |
| 无模块化管理 | JS 逻辑与 HTML 模板混合，难以复用和测试 |
| 静态资源路径混乱 | 图片、CSS 分散在各页面目录 |

### 2.2 功能缺陷

| 问题 | 位置 | 风险等级 |
|------|------|---------|
| 图表初始化脚本写在对应 DOM 元素之前 | `index.html` | **高** — 页面加载时直接报错，图表无法渲染 |
| CSS 语法错误：`cursor: none` 写在 `body {}` 外部 | `notice.html`、`people.html` | **中** — 样式失效，浏览器可能忽略后续样式规则 |
| 所有数据为 2024 年静态写死数据 | 全部页面 | **中** — 演示效果差，无法体现系统实时性 |
| 无搜索、筛选、分页 | 全部页面 | **高** — 不具备商用数据管理能力 |
| 无数据导出功能 | 全部页面 | **中** — 无法满足报表输出需求 |
| 无登录与权限控制 | 全部页面 | **高** — 商用系统必备安全机制缺失 |

### 2.3 技术债务

- 无构建工具，无法使用 ES Module、代码压缩、Tree Shaking
- 无依赖管理，ECharts、FontAwesome 通过 CDN 引入，版本不可控
- 无开发服务器，调试效率低
- 无路由管理，页面跳转通过 `<a href>`，体验差

## 五、核心功能实现说明

### 5.1 布局与导航组件化

**原项目问题**：每个 HTML 页面复制 100+ 行的 sidebar 导航代码。

**优化方案**：
- 提取 `AppLayout.vue` 作为唯一布局壳，内含 `AppSidebar` + `AppHeader` + `<router-view>`
- `AppSidebar.vue` 通过读取 `router.options.routes` 自动生成导航菜单，路由变更自动高亮
- 支持折叠/展开状态，由 `appStore.sidebarCollapsed` 统一管理
- 响应式设计：窄屏下自动折叠为图标模式

### 5.2 图表封装（ChartCard.vue）

**原项目问题**：每个页面独立编写 `echarts.init` + `window.addEventListener('resize')`，内存泄漏风险高。

**优化方案**：
- `ChartCard.vue` 统一封装 ECharts 生命周期：
  - `onMounted` 时 `echarts.init`
  - `watch` 监听 `option` prop 变化自动 `setOption`
  - `ResizeObserver` 监听容器尺寸变化自动 `resize`
  - `onBeforeUnmount` 时 `dispose` 释放实例
- 支持 `loading` prop，数据加载中显示骨架屏

### 5.3 数据表格封装（DataTable.vue）

**原项目问题**：无表格组件，数据以静态 HTML 展示。

**优化方案**：
- 基于 Element Plus `el-table` + `el-pagination` 封装
- 支持功能：
  - 搜索框实时过滤（前端过滤）
  - 自定义筛选插槽（状态筛选、类型筛选等）
  - 分页（前端分页，页码/页大小可调）
  - 导出 Excel（调用 `exportToExcel`，支持当前页/全部数据）
  - 行操作按钮插槽（查看详情、编辑、删除等）
  - 加载状态与空数据提示

### 5.4 Mock 数据与 API 预留

**设计原则**：Mock 数据格式与真实后端接口完全对齐，切换时无需修改业务代码。

**Mock 架构**：
```js
// mock/modules/device.js
Mock.mock('/api/device/list', 'get', (options) => {
  // 解析 URL 参数
  // 支持 keyword / type / status 筛选
  // 支持 page / pageSize 分页
  // 返回 { list, total }
})
```

**API 架构**：
```js
// api/modules/device.js
import request from '../request.js'

export const getDeviceList = (params) => request.get('/device/list', { params })
export const getDeviceStats = () => request.get('/device/stats')
export const getDeviceDetail = (id) => request.get(`/device/${id}`)
```

**切换真实 API**：
1. 移除 `main.js` 中的 `setupMock()` 调用
2. 配置 `VITE_API_BASE_URL` 环境变量指向真实后端地址
3. 确保后端接口路径与 Mock 定义一致（均以 `/api` 为前缀）

### 5.5 Axios 请求封装

**文件**：`src/api/request.js`

**特性**：
- 统一 baseURL（开发环境通过 Vite 代理 `/api` → `localhost:8080`）
- 请求头自动附加 `Authorization: Bearer <token>`
- GET 请求自动添加 `_t` 时间戳防止缓存
- 响应统一处理：
  - `code !== 200` 时自动 `ElMessage.error`
  - HTTP 状态码分类处理（401/403/404/500）
  - 网络错误友好提示

### 5.6 路由与权限

**文件**：`src/router/index.js`

**特性**：
- 10 个功能页面 + 登录页，全部懒加载（`() => import(...)`）
- 路由守卫 `beforeEach`：
  - 演示环境：自动调用 `authStore.setDemoUser()` 跳过登录
  - 生产环境：取消注释 `next('/login')` 启用真实登录校验
- 404 路由统一重定向到 Dashboard

### 5.7 状态管理（Pinia）

**appStore** (`src/stores/app.js`)：
- `sidebarCollapsed`：侧边栏折叠状态
- `fullscreen`：全屏状态
- `currentTime`：实时时钟（每秒更新，供 AppHeader 显示）
- `startClock()` / `stopClock()`：时钟生命周期管理

**authStore** (`src/stores/auth.js`)：
- `token` / `userInfo`：登录凭证与用户信息
- `setDemoUser()`：演示环境快速注入默认用户
- `login()` / `logout()`：登录登出方法（预留真实接口调用）

## 六、BUG 修复记录

| 序号 | 原问题 | 修复方式 | 验证方式 |
|------|--------|---------|---------|
| 1 | `index.html` 中 `echarts.init` 调用在 DOM 元素渲染之前 | 使用 Vue `onMounted` 生命周期，确保 DOM 就绪后再初始化图表 | 页面加载无控制台报错，图表正常渲染 |
| 2 | `notice.html`、`people.html` 中 `cursor: none` 和 `overflow-x: hidden` 写在 `body {}` 闭合大括号外部 | 迁移到 Vue SFC 的 `<style scoped>` 中，编译时自动检查 CSS 语法 | 构建通过，无 CSS 解析警告 |
| 3 | 所有数据为 2024 年静态写死数据 | MockJS 动态生成当前时间附近的数据，Dashboard 每 30 秒自动刷新 | 查看网络面板，每次刷新 `stats` 接口返回动态数值 |
| 4 | 页面跳转使用 `<a href>` 整页刷新 | 替换为 Vue Router `<router-link>`，单页无刷新切换 | 切换页面时无白屏，状态保持 |
| 5 | 无统一错误处理，接口失败时页面卡死 | Axios 拦截器统一捕获并 `ElMessage.error` 提示 | 模拟断网，页面显示友好错误提示 |

## 八、性能优化措施

| 措施 | 实现方式 | 效果 |
|------|---------|------|
| 路由懒加载 | `component: () => import('@/views/...')` | 首屏仅加载必要代码 |
| ECharts 按需引入 | 各页面独立 `import * as echarts from 'echarts'` | 避免全量引入，减少包体积 |
| 图表实例销毁 | `onBeforeUnmount` 中调用 `chartInstance.dispose()` | 防止内存泄漏 |
| ResizeObserver | 替代 `window.resize` 事件监听 | 仅监听图表容器，减少全局事件 |
| Mock 环境隔离 | `if (import.meta.env.DEV)` 条件加载 | 生产构建自动剔除 Mock 代码 |
| 请求缓存控制 | GET 请求自动附加 `_t` 时间戳 | 开发环境避免缓存干扰调试 |

## 十、扩展建议

1. **后端分页**：当前表格分页在前端完成，数据量超过 1000 条时建议改为后端分页，修改 `DataTable.vue` 支持 `server-side` 模式
2. **WebSocket 实时推送**：告警、监控画面可使用 WebSocket 替代轮询，减少服务器压力
3. **图表按需加载**：ECharts 完整包体积较大，可按图表类型按需引入 `echarts/charts`、`echarts/components`
4. **角色权限细化**：当前权限仅到登录层面，可扩展为 RBAC 模型（菜单级 + 按钮级权限）
5. **数据可视化大屏**：Dashboard 可进一步升级为 2K/4K 大屏展示，增加地图组件、数字翻牌器等特效
