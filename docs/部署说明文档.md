## 三、开发环境运行

### 3.1 启动开发服务器

```bash
npm run dev
```

### 3.2 访问项目

开发服务器默认运行在：

```
http://localhost:5173
```

如端口被占用，Vite 会自动尝试下一个可用端口（如 5174）。

### 3.3 开发服务器配置

`vite.config.js` 中的开发服务器配置：

```js
server: {
  port: 5173,
  proxy: {
    '/api': {
      target: 'http://localhost:8080',
      changeOrigin: true,
    },
  },
}
```

| 配置项 | 说明 |
|--------|------|
| `port` | 开发服务器端口 |
| `proxy['/api']` | 将 `/api` 开头的请求代理到后端服务 |
| `target` | 后端 API 地址（默认 `localhost:8080`） |

**修改后端地址**：

如需将代理指向其他后端地址，修改 `vite.config.js`：

```js
proxy: {
  '/api': {
    target: 'http://192.168.1.100:8080',  // 修改为目标地址
    changeOrigin: true,
  },
}
```

修改后需要重启开发服务器生效。

### 3.4 开发模式特性

- **HMR（热更新）**：修改代码后页面自动刷新，状态保持
- **Mock 数据**：自动加载 MockJS 模拟数据，无需后端即可调试
- **Source Map**：完整源码映射，便于浏览器调试

## 五、生产环境部署

### 5.1 Nginx 部署（推荐）

#### 5.1.1 安装 Nginx

```bash
# Ubuntu/Debian
sudo apt update
sudo apt install nginx

# CentOS/RHEL
sudo yum install nginx

# macOS
brew install nginx
```

#### 5.1.2 配置 Nginx

创建配置文件 `/etc/nginx/conf.d/smart-community.conf`：

```nginx
server {
    listen 80;
    server_name localhost;  # 修改为实际域名

    # 前端静态文件
    location / {
        root /var/www/smart-community/dist;  # 修改为实际构建输出路径
        index index.html;
        try_files $uri $uri/ /index.html;  # 支持 Vue Router history 模式
    }

    # API 代理
    location /api/ {
        proxy_pass http://localhost:8080/;  # 修改为实际后端地址
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # gzip 压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml;
}
```

> **重要**：`try_files $uri $uri/ /index.html;` 这行是必须的，否则刷新页面会 404。

#### 5.1.3 上传构建产物

```bash
# 将 dist 目录上传到服务器
scp -r dist/* user@server:/var/www/smart-community/dist/
```

#### 5.1.4 启动/重启 Nginx

```bash
# 检查配置是否正确
sudo nginx -t

# 重启 Nginx
sudo systemctl restart nginx
# 或
sudo nginx -s reload
```

#### 5.1.5 访问系统

```
http://your-domain.com
```

### 5.2 Docker 部署

#### 5.2.1 创建 Dockerfile

在项目根目录创建 `Dockerfile`：

```dockerfile
# 构建阶段
FROM node:18-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
RUN npm run build

# 运行阶段
FROM nginx:alpine
COPY --from=builder /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

#### 5.2.2 创建 nginx.conf

```nginx
server {
    listen 80;
    location / {
        root /usr/share/nginx/html;
        index index.html;
        try_files $uri $uri/ /index.html;
    }
    location /api/ {
        proxy_pass http://backend:8080/;
    }
}
```

#### 5.2.3 构建并运行

```bash
# 构建镜像
docker build -t smart-community-dashboard .

# 运行容器
docker run -d -p 80:80 --name smart-community smart-community-dashboard
```

### 5.3 静态文件服务器部署

如使用 Caddy、Apache 或其他服务器，核心要求：

1. 将 `dist/` 目录作为静态文件根目录
2. 配置所有路由都回退到 `index.html`（支持 History 路由模式）
3. 配置 `/api/` 路径代理到后端服务

## 六、Mock 切换真实 API

### 6.1 切换步骤

#### 步骤 1：关闭 Mock 拦截

编辑 `src/main.js`：

```js
// 修改前（开发模式）：
if (import.meta.env.DEV) {
  setupMock()
}

// 修改后（生产模式，完全关闭 Mock）：
// if (import.meta.env.DEV) {
//   setupMock()
// }
```

或者完全移除 Mock 相关代码：

```js
// 删除以下两行
import { setupMock } from './mock/index.js'
// setupMock()
```

#### 步骤 2：配置真实后端地址

**方式一：通过 Vite 代理（开发环境）**

修改 `vite.config.js`：

```js
server: {
  proxy: {
    '/api': {
      target: 'http://your-real-backend.com:8080',
      changeOrigin: true,
    },
  },
}
```

**方式二：通过环境变量（推荐）**

创建 `.env.development`：

```env
VITE_API_BASE_URL=/api
```

创建 `.env.production`：

```env
VITE_API_BASE_URL=https://api.your-domain.com/api
```

修改 `src/api/request.js`：

```js
const request = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL || '/api',
  timeout: 15000,
})
```

> 注意：环境变量必须以 `VITE_` 开头才能在客户端代码中访问。

#### 步骤 3：确保接口格式一致

真实后端接口需遵循以下规范：

- **请求路径**：与 Mock 定义完全一致（如 `/api/device/list`）
- **请求方法**：GET / POST / PUT / DELETE 与 Mock 一致
- **请求参数**：分页参数 `page`、`pageSize`，筛选参数格式一致
- **响应格式**：
  ```json
  {
    "code": 200,
    "message": "success",
    "data": { ... }
  }
  ```

### 6.2 接口对照表

| Mock 路径 | 方法 | 真实后端路径 | 说明 |
|-----------|------|-------------|------|
| `/api/dashboard/stats` | GET | 同左 | 社区概况统计 |
| `/api/device/list` | GET | 同左 | 设备列表（支持分页筛选） |
| `/api/alarm/list` | GET | 同左 | 告警列表 |
| `/api/alarm/:id/resolve` | POST | 同左 | 标记告警已处理 |
| `/api/people/list` | GET | 同左 | 人员列表 |
| `/api/house/list` | GET | 同左 | 房屋列表 |
| `/api/car/list` | GET | 同左 | 车辆记录 |
| `/api/patrol/list` | GET | 同左 | 巡更记录 |
| `/api/repair/list` | GET | 同左 | 报修列表 |
| `/api/repair/:id/status` | POST | 同左 | 更新报修状态 |
| `/api/notice/list` | GET | 同左 | 公告列表 |

> 完整接口列表请参见《优化说明技术文档》第七章。

### 6.3 验证切换结果

1. 重新构建项目：`npm run build`
2. 打开浏览器开发者工具（F12）→ Network 面板
3. 访问系统各页面

## 八、常见问题排查

### 8.1 构建失败

**现象**：`npm run build` 报错

**排查步骤**：
1. 检查 Node.js 版本：`node -v`（需 >= 18）
2. 删除 `node_modules` 重新安装：`rm -rf node_modules && npm install`
3. 检查 `package.json` 依赖是否完整

### 8.2 页面刷新 404

**现象**：点击导航正常，但刷新页面或直接从 URL 进入子页面时报 404

**原因**：Vue Router 使用 History 模式，需要服务器配置回退到 `index.html`

**解决**：
- **Nginx**：添加 `try_files $uri $uri/ /index.html;`
- **Apache**：在 `dist/` 目录创建 `.htaccess`：
  ```apache
  <IfModule mod_rewrite.c>
    RewriteEngine On
    RewriteBase /
    RewriteRule ^index\.html$ - [L]
    RewriteCond %{REQUEST_FILENAME} !-f
    RewriteCond %{REQUEST_FILENAME} !-d
    RewriteRule . /index.html [L]
  </IfModule>
  ```

### 8.3 接口请求跨域错误

**现象**：浏览器控制台报 `CORS policy` 错误

**解决**：
- 开发环境：通过 Vite 代理解决（已配置）
- 生产环境：后端服务配置 CORS 响应头，或前端 Nginx 统一代理

### 8.4 Mock 数据未生效

**现象**：开发环境下接口返回 404

**排查**：
1. 确认 `main.js` 中 `setupMock()` 被调用
2. 确认请求 URL 与 Mock 定义一致（包含 `/api` 前缀）
3. 检查浏览器控制台是否有 `[Mock] Mock service initialized` 日志

### 8.5 图表不显示或报错

**现象**：图表区域空白或控制台报错

**排查**：
1. 确认 ECharts 已正确安装：`npm ls echarts`
2. 检查图表容器是否有固定高度
3. 检查浏览器控制台是否有 ECharts 初始化错误

### 8.6 Element Plus 样式异常

**现象**：组件样式丢失或显示异常

**排查**：
1. 确认 `main.js` 中已导入 `element-plus/dist/index.css`
2. 检查是否有其他 CSS 覆盖 Element Plus 样式


版本号格式遵循 [SemVer](https://semver.org/lang/zh-CN/) 规范：`主版本.次版本.修订号`
